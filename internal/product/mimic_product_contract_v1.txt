MIMIC AI — PRODUCT CONTRACT (v1)

This document defines the non negotiable rules of the Mimic AI system.

----------------------------------------
1. CORE FUNCTION
----------------------------------------
Mimic translates HTML into Figma outputs using ONLY the user’s design system.

----------------------------------------
2. DESIGN SYSTEM RULE
----------------------------------------
- A DS component must ONLY be used if fully correct
- If uncertain → fallback to primitives
Incorrect DS usage is worse than fallback

----------------------------------------
3. GENERALIZATION RULE
----------------------------------------
Must work with ANY design system
Must NOT encode DS-specific assumptions or UI patterns

----------------------------------------
4. LEARNING RULE
----------------------------------------
Every build must improve:
- component resolution
- reduce fallbacks
- reduce tokens and MCP calls

----------------------------------------
5. CONTINUITY RULE
----------------------------------------
Learning persists and affects future builds

----------------------------------------
6. CORRECTION RULE
----------------------------------------
User corrections override and persist

----------------------------------------
7. ACCURACY RULE
----------------------------------------
Preserve:
- structure
- content (exact text)
Visual differences allowed if DS-aligned

----------------------------------------
8. PRODUCT BOUNDARY
----------------------------------------
End user sees only output, not internal mechanics

----------------------------------------
9. VALIDATION GATE
----------------------------------------
Every prompt must:
1. Improve HTML→Figma accuracy
2. Work for ANY DS
3. Improve learning
4. Avoid DS-specific assumptions
5. Avoid UI hardcoding
